1. Basics of Computer Network

The following exposition addresses the foundational principles of data communication architecture with rigorous adherence to established standards and a professional register.

1.1 Define a computer network. Why are networks important?

Definition: A computer network is a structured interconnection of autonomous computing devices (nodes) via a combination of hardware, transmission media, and software protocols for the express purpose of resource sharing, data exchange, and collaborative processing.

Importance and Justification:
The criticality of networks is predicated upon several socioeconomic and technical imperatives:

  • Resource Optimization: Networks facilitate the sharing of peripheral devices (e.g., high-volume printers, storage area networks) and computational power, yielding significant capital expenditure reductions and economies of scale.
  • Information Dissemination and Redundancy: They enable real-time data replication and distributed database management, ensuring high availability and fault tolerance through redundancy.
  • Communication Efficiency: Networks serve as the substrate for electronic mail, instant messaging, and Voice over IP (VoIP), collapsing temporal and geographic barriers and engendering a globalized operational environment.
  • Centralized Administration: They permit unified security policy enforcement, software deployment, and user authentication across an entire enterprise domain.

1.2 Explain the OSI Reference Model. List and describe the functions of each layer.

The Open Systems Interconnection (OSI) Reference Model, standardized as ISO/IEC 7498-1, constitutes a conceptual framework that partitions network communication into seven discrete strata of abstraction. Its primary function is to ensure interoperability between disparate systems and vendors.

Layer No. Layer Name Primary Functions and Description
7 Application Provides network services directly to the end-user application. It does not comprise the application itself (e.g., a web browser), but rather the protocols that enable the application to access the network (e.g., HTTP for web transfer). Functions include file transfer, email services, and directory services.
6 Presentation Concerned with the syntax and semantics of transmitted information. It translates data between the application format and a network-standard format. Core responsibilities include character code translation (e.g., ASCII to EBCDIC), data encryption/decryption, and compression/decompression.
5 Session Manages the dialog control between end-user application processes. It establishes, manages, and terminates communication sessions. It provides mechanisms for synchronization by inserting checkpoints into the data stream, allowing recovery from a disruption without restarting the entire transfer.
4 Transport Responsible for process-to-process delivery of the entire message. It ensures end-to-end reliability through error recovery, flow control, and segmentation/reassembly of data streams. This layer provides both connection-oriented (TCP) and connectionless (UDP) services, abstracting the network's physical topology from upper layers.
3 Network Governs logical addressing and routing. It determines the optimal physical path (routing) for data packets from source to destination across multiple intermediate networks (internetworking). The primary Protocol Data Unit (PDU) at this layer is the packet. It handles congestion control and logical addressing (e.g., IPv4, IPv6).
2 Data Link Provides node-to-node delivery across a single physical link. It transforms the raw transmission facility into a reliable link by framing data, detecting (and sometimes correcting) errors, and controlling media access (MAC). It is subdivided into Logical Link Control (LLC) and Media Access Control (MAC) sublayers. PDU: Frame.
1 Physical Defines the mechanical, electrical, functional, and procedural characteristics required to establish, maintain, and deactivate the physical link. It deals with the transmission of raw bit streams (PDU: Bits) over physical media, including voltage levels, data rates, cable specifications, and connector pinouts.

1.3 What is the TCP/IP model? How does it differ from the OSI model?

The Transmission Control Protocol/Internet Protocol (TCP/IP) model, also known as the Internet Protocol Suite, is the pragmatic, implementation-driven architectural blueprint upon which the global Internet is constructed.

Differences from the OSI Model:
The distinction between the two models is ontological rather than merely numeric. The OSI model is prescriptive (a protocol-independent standard for how communication should occur), whereas the TCP/IP model is descriptive (a protocol-dependent description of how the Internet does operate). Specifically:

  1. Genesis: OSI emerged from a formal standards committee (ISO) prior to protocol implementation. TCP/IP evolved from research (ARPANET) and was reverse-engineered into a model only after the protocols were operational.
  2. Layer Consolidation: The TCP/IP model amalgamates the OSI Presentation and Session layers into its single Application layer, reflecting a leaner, more vertical integration approach.

1.5 What are the advantages of using the layered approach in network design?

The layered architecture represents a foundational principle of modular software engineering applied to distributed systems. The advantages are manifold:

  1. Encapsulation and Abstraction: Each layer provides a defined set of services to the layer immediately above it while concealing the implementation details of how those services are performed. This reduces complexity and cognitive load on developers.
  2. Interoperability and Standardization: By defining standard interfaces between layers, equipment from different manufacturers can interoperate seamlessly. A Cisco router (Layer 3) can forward frames from an Intel Network Interface Card (Layer 2) precisely because the interface expectations are standardized.
  3. Modularity and Facilitated Innovation: Modifications or upgrades to one layer (e.g., moving from copper Ethernet to fiber optics at the Physical layer) are transparent to the upper layers. The TCP/IP stack continues to function identically regardless of the underlying transmission medium.
  4. Simplified Troubleshooting: Failure analysis proceeds via isolation. If a user cannot connect to a website, one tests Layer 3 (Can I ping the IP?), then Layer 4 (Is the port listening?), then Layer 7 (Is the service responding with correct HTTP codes?). This compartmentalization is indispensable for network forensics.

1.6 What is the main function of the Application layer in the OSI model? Provide examples of protocols operating at this layer.

Main Function:
The Application layer serves as the interface between the user's software process and the network stack. It provides the semantic meaning of the communication. Crucially, this layer is not the application executable (e.g., chrome.exe); rather, it is the suite of protocols that enable the application to request and interpret network data.

Protocol Exemplars (OSI Model):

  • HTTP/HTTPS: Hypertext Transfer Protocol (Secure) for web browsing.
  • SMTP: Simple Mail Transfer Protocol for email transmission.
  • IMAP/POP3: Internet Message Access Protocol / Post Office Protocol for email retrieval.
  • FTP: File Transfer Protocol for file repository access.
  • DNS: Domain Name System for name-to-address resolution.
  • SNMP: Simple Network Management Protocol for monitoring network devices.

1.7 What are the key differences between connection-oriented and connectionless communication in the Transport layer?

At the Transport Layer (Layer 4), the choice between connection-oriented and connectionless services dictates the Quality of Service (QoS) paradigm.

Feature Connection-Oriented (e.g., TCP) Connectionless (e.g., UDP)
Pre-communication Handshake Mandatory. A virtual circuit is established via a three-way handshake (SYN, SYN-ACK, ACK) before data exchange. None. Data transmission begins immediately.
Reliability High. Guarantees delivery through positive acknowledgments (ACK) and automatic retransmission of lost or corrupted segments. Unreliable. No guarantee of delivery, order, or duplicate protection.
Flow and Congestion Control Sophisticated algorithms (Sliding Window, Slow Start, AIMD) regulate transmission rate to match receiver capacity and network conditions. None. The sender may overwhelm the receiver or congest the network path.
Sequencing Strict. Segments are numbered and reordered at the receiver to reconstruct the original stream in the correct sequence. Unordered. Packets may arrive in any order; reassembly is left to the application logic.
Overhead Higher header size (20 bytes minimum) and state maintenance. Lower header size (8 bytes) and minimal state.
Use Case Applications requiring fidelity: File downloads, Email, Web browsing. Applications tolerant of loss but sensitive to latency: VoIP, Video Conferencing, DNS queries.

1.8 Explain the role of the Data Link layer in the OSI model.

The Data Link layer (Layer 2) serves a dualistic function of Media Access Control and Logical Link Control, effectively transforming the unreliable, raw bitstream of the Physical layer into a reliable, error-detected frame service for the Network layer.

Its specific responsibilities include:

  • Framing: Encapsulating the Network layer packet by adding a header and trailer to create a discrete unit of transmission called a frame.
  • Physical Addressing (MAC): Adding source and destination hardware addresses (MAC addresses) to the frame header to identify the specific interface on the local network segment.
  • Error Detection: Computing a checksum (CRC) appended to the frame trailer. The receiving node recalculates the checksum; a mismatch triggers frame discard (but typically not retransmission at this layer, as that is handled by Transport).
  • Flow Control: Preventing a fast transmitter from drowning a slow receiver on the same link (e.g., using PAUSE frames in Ethernet).
  • Access Control: Determining which device may transmit on a shared media at any given moment (e.g., CSMA/CD for legacy Ethernet, Token Passing for Token Ring).

1.9 What is multiplexing, and which layer in the OSI model is responsible for it?

Definition:
Multiplexing, in the context of network protocol stacks, is the process of combining multiple data streams or signals from different sources (or applications) into a single shared medium or channel. In the reverse direction, Demultiplexing is the process of separating the combined stream back into its constituent original parts and delivering them to the correct destination application.

Responsible Layers:
The concept of multiplexing manifests in distinct forms across multiple layers of the OSI model, each with a different scope:

  1. Transport Layer (Layer 4): This is the primary locus of logical multiplexing in network architecture. The Transport layer uses Port Numbers to differentiate between multiple concurrent communication sessions occurring on the same host. For example, a single IP address may simultaneously receive email (TCP Port 143) and web traffic (TCP Port 80). The Transport layer demultiplexes the incoming stream and delivers the segment payload to the correct socket/application process.
  2. Physical Layer (Layer 1): This refers to physical signal multiplexing. Techniques such as Frequency Division Multiplexing (FDM), Time Division Multiplexing (TDM), or Wavelength Division Multiplexing (WDM) combine multiple physical signals over a shared copper, fiber, or wireless frequency spectrum.

Conclusion: While the term is used in the physical sense at Layer 1, in the context of logical application-to-application communication referenced by this query, the Transport layer (Layer 4) bears the explicit responsibility for multiplexing and demultiplexing via port addressing.
